# 排课小程序 · 一期方案

交付形态：微信小程序
交付周期：7天-14天（考虑审核时间）
一期范围：老师端排课与学员管理

---

## 一、项目分期

考虑到交付周期与账号资质的办理节奏，项目分两期推进。

### 一期（本次交付）

面向老师的排课与学员管理工具，覆盖日常最高频的场景：课表、学员、课时、提醒。上线即可独立使用，不依赖学生端配合。

### 二期（需营业执照）

学生端预约、课包在线购买、课后笔记与作业、财务统计。

一期使用个人主体小程序，注册当天即可开始开发；二期需要企业主体以开通微信支付与教育类目，营业执照可在一期开发期间并行办理，不占用交付时间。

小程序主体支持从个人升级为企业，一期的代码与数据在二期可直接沿用。

---

## 二、关于 App

一期先做小程序，App 作为后续独立阶段规划。

主要考虑：

- App 上架国内应用商店需完成工信部备案，周期通常 2-4 周，超出本次交付窗口
- 苹果对功能较轻的应用有最低功能要求，纯排课工具存在被要求补充功能的可能
- 安卓需分别提交华为、小米、OPPO、vivo 等渠道，各自独立审核；后续每次更新也需重新过审

架构上，后端与数据层采用 Supabase，独立于前端实现。二期如开发 App，可以采用 Flutter，可直接复用现有数据库与业务规则，无需数据迁移，界面层为独立开发工作。

App 版本的启动需具备：营业执照、ICP 与 App 备案、苹果开发者账号（企业资质，
办理约 1-2 周）、各安卓渠道账号。这些为独立于开发的前置工作，建议在二期确定
后再行评估。

## 三、一期功能

### 学员管理

- 学员建档：姓名、手机号、级别标签、备注
- 学员详情：剩余课时、历史上课记录
- 课时充值：线下收款后手动录入

### 排课

- 周视图课表
- 新建单次课：选择学员、日期、时间、时长
- 新建循环课：每周固定时段，批量生成
- 编辑与取消单节课
- 时间冲突自动检测

### 上课记录

- 课程一键标记完成，自动扣减课时
- 缺勤标记，扣与不扣可选
- 课时变动全程留痕，可追溯

### 提醒

- 课前一小时提醒
- 学员课时余额不足提醒

### 首页

- 今日课程
- 本周课时统计
- 课时不足学员预警

## 四、技术方案

| 层 | 选型 |
|---|---|
| 前端 | Taro 3 + React + TypeScript |
| UI 组件 | NutUI + 自定义主题 |
| 后端 | Supabase |
| 数据库 | PostgreSQL |
| 登录 | 微信授权登录 |
| 提醒 | 微信订阅消息 + 定时任务 |

采用 Taro 是为保留后续编译 App 与 H5 的能力；采用 Supabase 是因其集成数据库、接口与定时任务，可在短周期内省去自建后端的时间。

课时数据采用流水记账方式，余额由变动记录累加得出而非直接修改。这样任何一次扣减都可回溯，避免出现课时对不上且无法查证的情况。

## 五、几点说明

一期界面文案按工具类定位撰写，使用「课表」「学员」「课时」等表述，暂不出现课程售卖相关内容，以确保审核顺利通过。二期待为企业主体后可放开。

提交审核提交安排，为可能的驳回重提预留三天余量。

若小程序注册出现意外阻碍，可先以 H5 形式交付，功能一致，添加至手机桌面即可使用，待账号就绪后再发布小程序版本。

微信平台的类目要求与主体升级规则可能调整，执行前以公众平台最新说明为准。
