# 订阅消息方案（第 1 步：模板申请清单 + 授权时机）

面向老师端的两类提醒：**课前提醒**、**课时不足**。接收人都是老师本人，在她自己手机上授权、在她手机上收到。

## 平台硬限制：一次性订阅

个人主体、教育工具类小程序**只能用「一次性订阅消息」**（长期订阅仅限政务/公共服务类目）。关键含义：

- **一次授权 = 该模板只能发 1 条**。要发多条，需多次授权累积额度。
- `wx.requestSubscribeMessage` 一次最多带 3 个模板；用户可勾「**总是保持以上选择，不再询问**」，之后再请求可静默累积额度。
- 但**我们无法读取用户是否勾了「总是保持」**。所以不能"每次进页面就静默请求"——对没勾的用户会反复弹窗，体验极差，可能直接导致老师关掉提醒。授权只在明确的用户动作后请求（见下）。
- 「课前提醒」这种高频场景，靠在几个时机反复请求累积额度，后端按**可用额度优先发最紧要的那条**。这是微信的限制，不是设计取舍。

## 一、需要申请的模板（2 个）

到 **小程序后台 → 订阅消息 → 公共模板库**，按下面搜索、选字段、拿模板 ID。

### 模板 1 · 课前提醒
- 搜索标题：`上课提醒` / `课程提醒` / `课程开始提醒`（选教育类目下最接近的）
- 选这几个字段（关键词）：

| 想展示 | 字段类型 | 内容示例 | 约束 |
|---|---|---|---|
| 上课时间 | `time` 时间 | 2026-07-22 15:00 | 标准时间格式 |
| 课程 | `thing` 事物 | 剑桥课程 / 补课 | ≤20 字符，区分剑桥/补课 |
| 学员 | `thing` 事物 | kk、王梓涵 等4人 | ≤20 字符，截断规则见下 |

> 不选「温馨提示」——「距上课1小时」是废话，触发时机本身已说明。
>
> **学员截断规则**：显示前 2 人姓名 +「等N人」，如 1 人「kk」、2 人「kk、王梓涵」、4 人「kk、王梓涵 等4人」，总长控制在 20 字符内。

### 模板 2 · 课时不足
- 搜索标题：`课时提醒` / `课时不足` / `余额不足提醒`
- 字段：

| 想展示 | 字段类型 | 内容示例 | 约束 |
|---|---|---|---|
| 学员姓名 | `thing` 事物 | 王梓涵 | ≤20 字符 |
| 剩余课时 | `number` 数字 | 2 | **优先选 number 型**，不要 thing 型 |
| 温馨提示 | `thing` 事物 | 课时不足，请及时续费 | ≤20 字符 |

> 公共模板的可选字段各不相同。**选定后把每个模板的「模板 ID」+「实际关键词 key」（后台会显示 `time3` / `thing2` / `number5` 这种）发我**，第 2 步按 key 映射数据。字段类型/字数不匹配会发送失败，以后台实际字段为准。

## 二、订阅授权时机（三个，均由明确用户动作触发）

1. **首页铃铃入口（主入口）**：点击首页右上角铃铛 → `wx.requestSubscribeMessage([课前提醒ID, 课时不足ID])`（两个一起，一个弹窗），引导勾「总是保持」。这是老师主动开启提醒的地方。
2. **建课成功后**：创建单次/循环课成功后，请求一次「课前提醒」，为新课攒额度。
3. **充值成功后**：充值成功后请求一次「课时不足」，为可能的余额提醒攒额度。

> **不做**「每次进首页静默补额」——拿不到「总是保持」状态，会对没勾的用户反复弹窗。

**循环课的额度局限（必须显式告知老师）**：循环课一次生成 N 节，但一次授权只攒 1 条额度，无法一次授权 N 条。所以循环课创建成功的 Toast 要说明：

> 「已生成 N 节课。提醒额度有限，将优先提醒最近的课程」

避免老师以为提醒功能坏了。

## 三、第 2 步会怎么发（仅预告，本步不做）

- **课前提醒**：云函数**定时触发器**（每 10–15 分钟）扫 `status=scheduled` 且 `startTime` 落在「未来 50–60 分钟」的课 → `cloud.openapi.subscribeMessage.send`（云环境免 access_token）发给老师 openid，消耗一条额度。按可用额度优先发最近的课。
- **课时不足**：扣课时写流水后检查余额，首次跌破 ≤2 时发；或每日扫一次兜底。
- 模板 ID 放 `src/constants/subscribe.ts`（前端请求授权用）+ 云函数配置（发送用），老师拿到 ID 后填。

## 注意

- **模拟器收不到订阅消息，只能真机测。**
- 需先在后台申请模板拿到 ID 才能实现发送逻辑。
